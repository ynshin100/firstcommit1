1타 2피의 결과물
=======
*웹(Web) 영역 개념정리!!!!
- 현재 우리가 작업하는 모든 노력 및 결과물
- Web을 통해서 보여지는 결과물
- Web Browser 통하지 않으면 여러분들이 작업한
결과물은 절대 사용자에게 제공될 수 없음!!!
- Web Browser : Chrome / Edge / opera / safari
- Client (Web Browser / request)
- NAVER (Portal Site)
- Server (response)

- Server는 Client의 요청에 따라서 response Message를 상황에 맞춰서 제공할 수 있다.
(*반드시 Server는 Client에게 하나 이상의 메세지를 전달해야만 합니다!! => Web 표준규약)

Message 종류
1) status 200 : 클라이언트도 정상적으로 서버에 요청했고, 서버도 클라이언트의 요청에 정상적으로 피드백을 했다는 의미를 담고있는 메세지

2) status 303 : 클라이언트의 정상적인 요청에 서버가 다른 대안을 갖고있는 웹 페이지로 리다이렉션을 진행한 경우

3) status 404 : 클라이언트가 존재하지않는 도메인, 경로 URL을 서버측에 무리하게 요청한 경우

4) status 500 : 클라이언트는 서버측에 정상적으로 데이터 요청을했으나, 서버에서 해당 데이터를 피드백할 수 없는 불가피한 상황 (*서버가 다운 / 폭주 / 해킹)
