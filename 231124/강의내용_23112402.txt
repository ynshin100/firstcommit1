이터러블 객체 : 배열, 메서드

이터러블 객체 : 
1) for of 반복문
2) 전개연산자 구문 (...)
3) 구조분해할당

이터러블 객체는 유일무이한 기능을 구현할 수 있는 Symbol 함수를 가지고 있는데, 해당 Symbol 함수는 "이터레이터"라는 심볼키로 구성되어있다

배열이름[심볼키이름]( )

Array Iterator { }

ƒ next( )

배열이름.next( )

이터러블 객체안에 담긴 이터레이터 객체라는 심볼키는 next( ) 메서드 함수를 가지고 있기 때문에 => 이터러블 객체는 next( ) 메서드 함수를 사용할 수 있다!!!

프로토타입 함수 : Array( )

인스턴스 객체 (*배열 = 이터러블한 객체) : arr

그렇다면, 이 좋은 next( ) 메서드 함수 => 이터러블 객체말고는 사용이 불가한것인가?

제너레이터 객체를 생성하면, 이터러블 객체의 전유물인 next( )메서드 함수를 동일하게 사용할 수 있다!!!!
